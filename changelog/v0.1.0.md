# v0.1.0 - Initial Production Release

This release marks the transition from a prototype to a production-ready, header-only utility library for 3D coordinate system transformations.

## Key Changes

### Core Library & Architecture
- **Header-Only Split**: Logically split the monolithic core into smaller, focused headers: `types.h`, `ratio.h`, `meta.h`, `converter.h`, and `presets.h`.
- **Class-Level Precision**: Templated `TransformationConverter<T>` for consistent precision (float/double) across operations.
- **Improved Determination of Handedness**: Refactored the handedness logic into a pure, `static constexpr` algorithm used during construction.
- **Stateless & Immutable**: Removed stateful caching from metadata, ensuring pure value-type behavior.
- **Zero Overhead**: Fully header-only architecture with optimized mapping logic and return-by-value overloads.

### Developer Experience (DX)
- **TransformationMetaBuilder**: A fluent builder for expressive coordinate system definitions.
- **Custom Literals**: Added intuitive string literals like `"X+"_a` and `"Y-"_a` for defining axis alignments.
- **Improved Error Messages**: Implemented `static_assert` inside the `Traits` system to provide clear, actionable compile-time errors for missing specializations.
- **Common Presets**: Added `Presets` namespace with factory methods for `Unity()`, `Unreal()`, `ROS()`, and `OpenGL()`.

### Performance & Flexibility
- **Batch Conversions**: Introduced `convert_points` and `convert_sizes` using `std::span` for efficient batch processing and compiler auto-vectorization.
- **No-Exceptions Mode**: Added `TRANSFORMATION_NO_EXCEPTIONS` support for environments where exceptions are forbidden (replaces `throw` with `assert`).

### Build & Integration
- **Namespaced Targets**: Migrated to a cleaner `Transformation::Core` and `Transformation::Plugins` naming convention.
- **vcpkg Port**: Created a full `vcpkg` port with optional features for Eigen and PCL support.
- **Configured Headers**: Implemented `plugins.h.in` to bake build-time plugin selection directly into the installed headers using numeric values (`#if 1/0`) to avoid global macro clashes (`USE_EIGEN`, `USE_PCL`).
- **CI/CD**: Added a GitHub Actions workflow for automatic Doxygen documentation deployment to GitHub Pages.

### Plugins
- **Eigen & PCL Support**: Specialized traits for seamless integration with industry-standard math libraries.

### Testing
- Comprehensive test suite including exhaustive handedness validation, double-precision checks, and batch conversion verification.
